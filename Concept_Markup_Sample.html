Cloudy: An Overview

Cloudy is a private cloud ecosystem of unique servers and applications. Cloudy provides application deployment flexibility, streamlined hardware resource use, and a common hosting environment.

Cloudy is equipped with Weather Status, a user interface (UI) solution that enables entry-level admins to maintain Cloudy and its corresponding application load without knowledge of command-line code.

The Hub 1, Hub 2, and Hub 3 systems provide servers for the Cloudy ecosystem. All systems that contribute servers to Cloudy can run select Cloudy-approved applications. For Cloudy 1.1 (Weather Status Version 2.1), the Cloudy server contribution and application distribution is as follows:

Cloudy Basics: Kubernetes 101

Cloudy is based on Kubernetes, a FOSS (Free Open Source Software) Orchestration Tool designed to manage containerized applications across multiple hosts. This means admins can deploy, host, update, and maintain applications and their availability 24/7. For example, if Appy runs on a server that needs maintenance, end users can still experience this application seamlessly during server repairs since the instance of this software can be transferred to an alternate Cloudy server.

Key terms admins will encounter while interacting with/and or observing Cloudy via Weather Status are defined, below:

  • Cloudy Applications: These are pre-approved applications designated to run within the ecosystem. They can be scheduled to run on any Cloudy worker node.
  • Node: The terms "node" and "server" can be used interchangeably.
  • Worker Node: A worker node is a server that provides its resources to Cloudy for application scheduling and hosting.
  • Cluster: A cluster is a group of servers.
  • Container: This is an executable image containing software and all its code libraries.
  • Control Plane Node: A control plane node (aka, an orchestrator node) is a server running an instance of the software library responsible for managing and deploying containers in Cloudy. There are two (2) control plane nodes within Cloudy, indicated by a "C" icon in Weather Status.
  • Cordon: This action marks a worker node as unusable to the control plane node(s). When a worker node is marked as "cordoned," the control plane node will not schedule applications to run on the designated node.
  • Drain: This action cordons a worker node and also forces a removal of all applications running on it. Once a worker node has been drained, the control plane node will then attempt to reschedule the applications on another available worker node.


Application Priority

When managing applications within Cloudy, the following priority scheme should be taken into account:

Application Priority Notes
Appy 1 This is an essential service. Appy must be maintained 24/7.
Happy 2 This is an essential service. Happy must be maintained 24/7.
Chappy 3 This is an optional service. Chappy should be maintained under ideal conditions.
Sappy 3 This is an optional service. Sappy should be maintained under ideal conditions.
Flappy 3 This is an optional service. Flappy should be maintained under ideal conditions.
